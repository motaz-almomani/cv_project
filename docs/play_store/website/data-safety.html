Collected Data types below are processed by the current app design (Firebase Auth + Firestore + optional local preferences).

Overview

Does the app collect or share user data? Yes — data you provide for account creation and résumé content is stored using Firebase Auth and Cloud Firestore. Treat Firebase/Google as a service provider for hosting and authentication.
Is all data encrypted in transit? Yes — HTTPS/TLS between the app and Google’s services (standard Firebase SDK behaviour).
Can users request data deletion? Yes — see Account & data deletion and Privacy Policy §6–8.

Data collected

Personal info

NameOptional display name stored in Firebase user profile.
Email addressCollected for authentication; stored by Firebase Authentication.
Other user-generated CV contentIncludes title, bio/summary, job history, education, skills, certificates, links, phone, address, and similar fields you enter into CV forms; stored in Firestore under your user ID.

App activity & diagnostics (minimal)

App interactions / crash logs Not collected by custom code in the baseline description unless you add Analytics or Crashlytics. If you add them, declare them here and in Play Console.

Device or other IDs

Firebase user ID (UID) Generated by Firebase Auth to identify your account and link Firestore documents — not an advertising ID.

Purpose of collection (per data type)

Account managementEmail, UID, password handling — sign-in, account security, password reset flow (via Firebase).
App functionalityStoring, editing, listing, and exporting résumés; associating CVs with the signed-in account.

Data sharing

Third parties Firebase / Google operates the authentication and database layer. Data is processed under your Firebase configuration; it is not “sold”. Do not declare unrelated ad-tech sharing unless you integrate it.

Optional: Data uploaded to Firebase (Firestore)

Cloud storage / database User-provided CV records are stored in Cloud Firestore (not Cloud Storage for files in the default code path unless you add it). Declaring “Photos and videos” or “Files” in Play Console is only needed if your shipped app uploads those categories.

Encryption & security (Play-aligned summary)

  • In transit: Firebase client SDKs use encrypted connections.
  • At rest: Google encrypts Firebase data at rest on its infrastructure.
  • Access control: Developers must configure Firestore Security Rules so users can access only their own documents (verified by UID).

Data deletion

  • Users can request deletion by email — see Deletion policy.
  • Firestore documents tied to userId should be deleted when the account is removed (manually, admin script, or Cloud Function).
  • Local-only preferences may remain until app data is cleared or the app is uninstalled.
Play Console checklist: For each data type you mark as collected, declare purposes (e.g. account management, app functionality), whether collection is optional/required, and whether data is encrypted in transit. This HTML is a drafting aid — the form in Console is authoritative.
